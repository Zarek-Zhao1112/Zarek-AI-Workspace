# Writing Rule

> 日报/周报写作规范，让AI生成高质量的工作汇报。

---

## 核心原则

### ❌ 不要写成流水账

```
今天做了Seller分析
修改了代码
测试了功能
```

### ✅ 要体现业务价值

```
分析了10个异常Seller的RMA数据
→ 发现3个Seller存在高退货率问题
→ 下一步：与导师沟通，制定改进方案
```

---

## 三段式结构

每条日报必须包含：

```
工作内容 → 业务价值 → 下一步
```

| 部分 | 说明 | 示例用词 |
|------|------|----------|
| 工作内容 | 做了什么具体的事 | 分析、建立、完善、优化、设计、沉淀 |
| 业务价值 | 为什么做、解决什么问题、有什么用 | 发现了、解决了、提升了、降低了 |
| 下一步 | 明天/后续要做什么 | 计划、准备、跟进、推进 |

---

## 推荐用词

### ✅ 推荐动词

| 类型 | 推荐用词 |
|------|----------|
| 分析类 | 分析、梳理、诊断、排查、对比 |
| 建设类 | 建立、搭建、设计、开发、实现 |
| 优化类 | 优化、完善、改进、迭代、升级 |
| 沉淀类 | 沉淀、总结、归档、文档化、标准化 |

### ❌ 禁用词汇

| 禁用 | 替换为 |
|------|--------|
| 搞 | 开发、实现、完成 |
| 弄 | 处理、配置、设置 |
| 修 | 优化、修复、调整 |
| 写代码 | 开发、实现、构建 |

---

## 老板视角

### 为什么做（Why）

- 这个任务解决什么业务问题？
- 不做会有什么后果？

### 有什么价值（Value）

- 做完之后能带来什么好处？
- 对团队/公司有什么贡献？

### 以后有什么用（Future）

- 这个成果能否复用？
- 后续还能怎么扩展？

## 日报模板

### 模板一：功能开发类

```
【工作内容】
完成了XX功能的开发与测试

【业务价值】
解决了XX问题，预计可以提升XX效率XX%

【下一步】
计划完成XX，预计XX时间完成
```

### 模板二：数据分析类

```
【工作内容】
分析了XX数据，共处理XX条记录

【业务价值】
发现XX问题，识别出XX个异常项，为XX决策提供数据支撑

【下一步】
跟进XX问题，与XX沟通解决方案
```

### 模板三：问题解决类

```
【工作内容】
排查并解决了XX问题

【业务价值】
消除了XX风险，保障了XX正常运行

【下一步】
沉淀解决方案，完善XX文档
```

---

## 日报格式示例

```markdown
## 2026年6月29日 工作日报

### 今日完成

1. 【Seller分析】分析了10个异常Seller的RMA数据，发现3个Seller存在高退货率问题（>15%），已标记为重点关注对象
   - 业务价值：提前识别风险Seller，避免平台罚款
   - 下一步：与导师沟通，制定Seller改进方案

2. 【AI平台】优化了Seller健康度评分算法，新增库存周转率指标
   - 业务价值：评分更全面，能更准确识别Seller问题
   - 下一步：测试新算法，收集反馈

### 明日计划

1. 完成Seller改进方案初稿
2. 测试AI招商助手的新功能

### 问题/阻塞

1. 某个Seller数据接口异常，已提交IT工单
```

## 周报模板

```markdown
## 第XX周 工作周报

### 本周完成

1. 【项目进展】完成了XX功能开发，解决了XX问题
2. 【数据分析】分析了XX数据，发现XX问题
3. 【问题解决】排查并解决了XX问题

### 本周成果

| 成果 | 价值 |
|------|------|
| XX功能上线 | 提升XX效率XX% |
| XX问题解决 | 降低XX风险 |

### 下周计划

1. 完成XX功能开发
2. 分析XX数据
3. 跟进XX问题

### 问题/需要支持

1. XX问题需要导师指导
```
