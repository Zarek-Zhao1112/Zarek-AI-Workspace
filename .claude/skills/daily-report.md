# Daily Report Skill

> 日报生成规范 - AI协作标准流程

---

## 触发条件

用户提供工作内容（任意格式）后，自动触发日报生成流程。

---

## 输出要求

每次生成日报必须包含两个版本，用分隔符隔开：

### 分析版（内部参考）

**用途：** 识别新技能、更新画像

**结构：**
1. **今日工作内容** - 列出具体工作
2. **画像更新** - 识别新增/强化的能力，填入表格（能力 | 说明）

**分析维度：**
- 从工作中提取可量化的能力点
- 判断是否为新技能或现有技能的强化
- 记录具体实例支撑

### 复制版（直接粘贴）

**用途：** 粘贴到日报系统

**格式：** 纯文本，逗号隔开

**每行结构：** 工作，业务价值，经验沉淀

**示例：**
```
参加周会明确AM日常工作，掌握PO控库存/竞品调价/BD抓新品三项核心职责，业务理解力提升
```

**注意事项：**
- 去掉"业务价值""下一步""问题解决"等标签词
- 简洁专业，可直接粘贴
- 使用推荐词汇：分析、建立、完善、优化、设计、沉淀
- 禁用词汇：搞、弄、修、写代码

---

## 文件更新规则

### ABOUT_ME.md

**更新位置：** 优势表格

**更新逻辑：**
- 新增能力行（如果是新技能）
- 或更新实例列（如果是现有技能的强化）

**命名规范：**
- 能力名称：XX能力（4-8字）
- 说明：能XX（动词开头）
- 实例：完成/掌握/学习XX（具体事项）

### INTERNSHIP.md

**更新位置：** 今日工作记录

**格式：**
```markdown
### YYYY年M月D日（入职第X天）

| 工作 | 业务价值 | 经验沉淀 |
|------|----------|----------|
| 工作1 | 业务价值1 | 经验沉淀1 |
```

**注意事项：**
- 日期格式：YYYY年M月D日（入职第X天）
- 每条工作对应一行
- 业务价值：说明对业务的贡献
- 经验沉淀：说明个人能力提升

## Git提交规范

**提交时机：** 用户确认后自动提交

**提交格式：**
```
feat: 更新日报 - M.D主题概述
```

**示例：**
```
feat: 更新日报 - 7.1周会明确AM职责，新增5项能力画像
```

**推送：** 自动推送到 GitHub

---

## 特殊规则

1. **日报内容不上传为文件** - 仅用于学习和更新画像
2. **先看再推** - 生成后先展示，确认后再推送
3. **三角度分析** - 导师/HR/老板视角（分析版可选）
4. **体现业务价值** - 不要写成流水账
