## AI辅助开发数据安全规范

> 公司明确：销售数据等为隐私数据，不能交给AI分析。

### 敏感数据分类

| 敏感等级 | 数据类型 | 存储位置 |
|----------|----------|----------|
| 🔴 高 | 卖家GMV、毛利、RMA% | `data/seller_history/*.json` |
| 🔴 高 | SKU销售明细 | `data/sku_analysis/**/*.json` |
| 🔴 高 | 企业微信聊天记录 | 用户本地/页面输入 |
| 🟡 中 | 问题记录、联系人信息 | `data/*.xlsx` |
| 🟢 低 | 品牌官网公开信息 | 外部网站 |

### AI调用风险清单

| 页面 | AI功能 | 风险 |
|------|--------|------|
| 卖家分析 | 无 | ✅ 完全安全 |
| 商家通讯录 | 无 | ✅ 完全安全 |
| 运营总览 | 无 | ✅ 完全安全 |
| 品牌线索 | 品牌信息分析 | 🟡 发送公开信息 |
| 招商管理 | AI邮件生成 | 🟡 发送品牌公开信息 |
| 问题管理 | 聊天记录提取、AI回复 | 🔴 发送聊天记录 |

### AI辅助开发原则

1. **读取代码文件** → 安全，不含业务数据
2. **读取Excel/JSON数据文件** → 需提醒用户数据会经过AI
3. **编写涉及外部API的功能** → 必须说明数据流向
4. **卖家分析页面** → 100%本地计算，可放心使用

### 关键认知

- **AI工具是双刃剑**：提升效率的同时，要注意数据隐私
- **区分代码和数据**：读取.py文件安全，读取.xlsx/.json需谨慎
- **本地优先**：能本地处理的数据，不要发送到外部AI服务
